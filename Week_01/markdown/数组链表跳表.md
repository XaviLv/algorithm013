# 数组/链表/跳表

## 数组 Array

内存管理器，开辟了一段连续内存

适合：需要频繁通过下标访问元素的情况；
不适合：有大量添加删除操作的时候。

### 优点

- 随机访问元素，时间复杂度 O(1)

### 缺点

- 插入删除操作，时间复杂度 O(n)

## 链表 Linked List

### 优点

- 添加删除操作，时间复杂度 O(1)

### 缺点

- 随机访问元素，时间复杂度 O(n)

## skip list 跳表

思想：升维，空间换时间

Redis、LevelDB 使用跳表代替平衡树

### 限制：只能用于元素有序的情况

### 缺点：索引维护成本较高
